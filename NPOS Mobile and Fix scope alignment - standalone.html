Latest Update · Change Log Alignment Meeting · Jul 7, 2026

Based on the Jul 7 alignment meeting, this roadmap has been updated to formally scope the program for mainland China and to adjust / add three requirements. Details below.

Scope
Title updated to "Mainland China · Four‑Phase Features Delivery Roadmap" — confirming the program's geographic scope is mainland China (MLC).
Added
Phase 02 · NPOS Mobile Scale — added NBY Product Sales Rollout (Scale Must). Brings the pilot's NBY capability to all 200+ scale stores; pairs with NBY Price Override to cover the full 4 GTINs × 20–30 price‑point ladder.
Added

Moved

In some countries we allow returns even if the consumer lost their receipt, is not a member (so we cannot find the order in Member Order History), and the order isn't in the Electronic Journal (EJ) either — there's no way to verify the consumer's original order.

For an unverified return we can still add the item to the cart and use the lowest price of that item in the last 6 months. For unverified returns the only refund tender we allow is gift card.

ReturnRealize After‑Exchange ReturnSolution to handle a return of an item that was previously exchanged.
View source · GC NDP Confluence
GC Priority · P0Global Feedback · P0

Solution to handle a return of an item that was exchanged. It's a problem in 1POS today: if a consumer on visit 1 buys an item, comes back on visit 2 to exchange it, and then on visit 3 decides to return that exchanged item — we don't have any payment details from visit 2 because the original payment was taken in visit 1.

1POS today only allows refund to a gift card in this case; if the consumer wants cash or bank card refund, 1POS does a tender exchange: exchange that gift card for cash, then refund to the bank card.

This quarter we're trying to find a better solution than tender exchange. If not, we'll build tender exchange. (Global double‑confirm needed.)

Promotion Flexibility — Greater China (MLC, HK, TW) Item‑Level Discount Stacking. GC needs the promotion engine to natively stack ≥ 2 discount rules at item level, eliminating manual reprice across MLC · HK · MC · TW.

Current state: TW has already proven the two‑layer model (L1 base auto + L2 booster manual), but MLC's 200+ doors are blocked at the system layer — the pricing engine only supports a single base discount today.

Evidence (same SKU pool · WK52, TW 10 doors vs MLC 200+ doors):

· FTW — TW Final MKD% −3.0 pts vs MLC, AUR +$7.4 → margin protection
· APP‑Pant — TW Final MKD% +10.4 pts, WOS ~8 wk faster → accelerated liquidation

EO: ~$5M / year on FTW alone (AUR Gap $7.4 × ~55K units; 2 stacking events / month × 3‑day weekend). Broader upside on APP & seasonal categories.

Next step (from one‑pager): Global team to assess feasibility and propose a solution for item‑level multi‑layer discount stacking across MLC · HK · MC · TW.
