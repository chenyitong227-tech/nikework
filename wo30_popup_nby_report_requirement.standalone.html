2设计原则

  • POP UP 不新增独立 Tab：通过新增全局业务类型筛选器（母店常规 @Wo / POP UP @Wo / All）+ 在现有 Tab 中扩展 POP UP 特有字段（POP UP Inv / shipped ST% / Event / Period）实现。
  • 新增 NBY Tracking 独立 Tab：用于独立 tracking NBY 业务（小物 + driven baseline 产品），仅在 NSO 范围下可用。
  • 业务类型可区分：母店常规 @Wo / POP UP @Wo / All — 在 6 个标准 Tab 上统一支持。
  • KPI 一致性：任何 KPI 的删除 / 新增 / 调整，需在所有视图前后保持一致。
  • 可下载：Raw Data 下载 Excel 需带业务线 & NBY 分类的 flag，便于业务自主分析。

14权限控制

业务范围权限控制规则
NSO 店铺 每家 NSO 店铺设置一个独立的 AD group。用户获得某家 NSO 店铺的 AD group 权限后，即可查看该店 POP UP 和 母店 @Wo 3.0 的全部数据。
NVS Follow 现有逻辑，暂不改动。
权限控制按 AD group 颗粒度实现，覆盖 Overview / Product / Raw Data / Weekly Report / SPH Table / Order Fulfillment / NBY Tracking 等所有 Tab；用户在已授权范围内可正常使用业务类型、Door、Period、NBY 类型等筛选器。
